Journée d'intégration

Développement d'une application web PHP MVC pour la gestion des ateliers de la journée d'intégration PICNIC du BTS SIO — puis refactoring complet vers le patron de conception DAO avec tests unitaires PHPUnit et documentation phpDocumentor.

Contexte

Le BTS SIO du lycée Saint-John Perse de Pau organise chaque année une journée d'intégration PICNIC pour les nouveaux étudiants. L'objectif est de les accueillir et de leur permettre de se retrouver autour d'activités communes : théâtre, algorithmes déconnectés, entretiens d'embauche simulés, escape game.

Dans ce cadre, l'atelier consistait à développer une application web PHP pour afficher et gérer les ateliers proposés lors de cette journée, en suivant une architecture MVC claire, puis en faisant évoluer le code vers un patron DAO plus robuste et maintenable.

Mission & Objectifs

Concevoir une application PHP en architecture MVC permettant d'afficher les ateliers de la journée d'intégration à partir d'un fichier XML, puis refactorer cette application en appliquant le patron de conception DAO pour séparer l'accès aux données de la logique métier — avec mise en place de tests unitaires PHPUnit et génération automatique de la documentation via phpDocumentor.

Réalisations

  • V1 — Application MVC PHP — architecture Modèle-Vue-Contrôleur avec routage par paramètre GET (?controleur=gestionAteliers&action=consultation), contrôleurs gestionAteliers.php et gestionGlobale.php
  • Lecture XML avec SimpleXML — classe FichierXML lisant un fichier ateliers.xml contenant les 4 ateliers PICNIC (Théâtre, Algorithmes déconnectés, Entretiens d'embauche, Escape game) et retournant les données à la vue
  • Connexion PDO à la base de données — classe Base encapsulant la connexion PDO avec gestion des erreurs (PDOException), requêtes SQL paramétrées, table Developpeur (id, nom, prénom)
  • V2 — Refactoring DAO — patron Data Access Object : classe entité Developpeur avec getters typés, BaseDAO générique, BaseDeveloppeurDAO spécialisée avec gestion des droits d'accès restreints selon le rôle (lecture/écriture)
  • Tests unitaires PHPUnit — fichiers de test pour valider le comportement des modèles, lancement via vendor/bin/phpunit --testdox tests/, structure séparée tests/modele/ et tests/controleur/
  • Documentation phpDocumentor — commentaires PHPDoc sur toutes les classes et méthodes, génération automatique de la documentation HTML avec php phpDocumentor.phar run -d ./site -t ./documentation
  • Environnement GitHub Codespace — configuration Docker + MariaDB, scripts bash d'initialisation, rechargement et sauvegarde de la BDD (initBDD.sh, reloadBDD.sh, saveBDD.sh), intégration XDebug pour le débogage pas à pas

Extraits de code

Classe FichierXML — lecture des ateliers depuis le fichier XML (V1)

/** * Classe qui permet d'extraire les données des ateliers * stockées dans un fichier XML */ class FichierXML { private $file; public function __construct($nomFichier = 'ateliers.xml') { $this->setFile('modeles/' . $nomFichier); } /** * Retourne la liste des ateliers depuis le fichier XML * @return SimpleXMLElement */ public function getLesAteliers() { try { $contenu = simplexml_load_file($this->file); } catch (Exception $e) { echo $e->getMessage(); } return $contenu; } }

Classe BaseDeveloppeurDAO — accès DAO avec droits restreints selon le rôle (V2)

class BaseDeveloppeurDAO extends BaseDAO { /** * Connexion à la BDD avec les habilitations adaptées à l'action */ private function setConnexionSelonRole(string $role) { $this->setConnexionBase( $_ENV['local_dsn'], $_ENV[$role], $_ENV['pwd' . $role], $_ENV['options'] ); } /** * Récupère tous les développeurs — retourne un tableau d'objets Developpeur * @return Developpeur[] */ public function getLesDeveloppeurs(): array { $this->setConnexionSelonRole("DevRead"); $resultat = $this->query("SELECT * FROM Developpeur;")->fetchAll(); $objets = []; foreach ($resultat as $ligne) { $objets[] = new Developpeur( (int)$ligne['id'], (string)$ligne['nom'], (string)$ligne['prenom'] ); } $resultat = null; // fermeture connexion return $objets; } }

Cet atelier m'a permis de consolider l'architecture MVC en PHP et de découvrir le patron DAO, qui sépare nettement la logique d'accès aux données de la logique métier. La gestion des droits d'accès à la base de données selon le rôle (lecture seule via JI_Dev_Read) m'a sensibilisée aux bonnes pratiques de sécurité dès la conception.

La mise en place de tests unitaires PHPUnit et la génération de documentation phpDocumentor m'ont donné une vision complète du cycle de développement professionnel : conception, codage, test et documentation technique.
